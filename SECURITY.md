# Security Policy

## Supported Content

This repository contains documentation, workflow guidance, and lightweight
utility scripts. It does not currently distribute production services or
runtime systems.

## Reporting a Vulnerability

If you find a security issue in scripts, examples, or contributed workflow
material, please open a private report through the repository host if available.
If private reporting is not available, open an issue with only high-level impact
details and avoid publishing exploit steps.

Include:

- Affected file or workflow.
- Expected impact.
- Steps to reproduce, when safe to share privately.
- Suggested mitigation, if known.

## Data Safety

Sports biomechanics data may include identifiable athlete, patient, or
participant information. Contributions should not include private motion
capture, video, medical, or performance data unless it is explicitly public,
properly licensed, and de-identified.

Do not commit secrets, API keys, proprietary datasets, license files, or
commercial software credentials.
